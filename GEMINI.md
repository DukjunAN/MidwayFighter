# MidwayFighter 프로젝트 가이드라인

이 파일은 MidwayFighter 프로젝트의 핵심 원칙과 설정을 기록합니다.

## 1. 기본 설정 (Core Settings)
- **전진 방향**: Three.js 표준인 `-Z`축을 앞방향(Forward)으로 사용합니다.
- **렌더링**: 유니티 스타일의 과노출(Whiteout)을 방지하기 위해 다음 설정을 고수합니다.
  - `renderer.toneMapping = THREE.ACESFilmicToneMapping`
  - `renderer.toneMappingExposure = 0.5`
- **배경**: `Sky.js`와 `Water.js`를 활용하여 '6월 4일 오전의 맑은 바다'를 구현합니다.

## 2. 개발 원칙 (Development Principles)
- **자산 보존(Asset Preservation)**: 이전에 완료된 동작, 모델 크기, 기믹 설정은 절대 임의로 변경하지 않습니다. 명시적 요청이 없는 한 '스스로의 판단'에 의한 최적화나 수정을 금지합니다.
- **사전 문의 의무**: 요청된 작업과 관련된 부분이라 하더라도 기존 자산의 변경이 발생할 가능성이 있다면, 반드시 사용자에게 해당 내용을 강조하여 문의하고 승인을 얻어야 합니다.

## 3. 골든 세팅 (Golden Settings - 절대 보존)
다음 설정값들은 사용자가 최종 확정한 값으로, 명시적 요청 없이는 절대 수정할 수 없습니다.

### 비행 물리 및 속도 (2026-05-07 업데이트)
- **보간 계수(Lerp)**: `0.01` (기존 0.02에서 더 둔화, 극한의 묵직한 기동)
- **최대 뱅크(Roll)**: `60도` (`Math.PI / 3`)
- **최대 피치(Pitch)**: `30도` (`Math.PI / 6`) - **최대 가동 범위 조정**
- **회전 순서**: `YXZ` (피칭 시 수평 유지 필수)
- **Yaw 로직**: `currentYaw -= currentRoll * 0.01` (Roll 비례 선회)
- **속도 스펙**: 최저 `130`, 평균 `250`, 최고 `512` (km/h)
- **부스트 제한**: 최고속 `3초` 유지 시 `평균속도`로 강제 회귀 (과열 시스템)

### 조종 UI (ControlUI)
- **조이스틱**: 
    - 위치: `Pos(0, -9.7, -25.6)`
    - 방향: `Rot(40, -91, 0)` (Degree 단위 초기값)
    - 크기: `Scale(7.11)`
    - 가동 범위: 당기기(Pitch Up) `-40도`, 밀기(Pitch Down) `+50도` (RZ축)
- **스로틀(Throttle.glb)**:
    - 위치: `Pos(-31, -4, 13)`
    - 방향: `Rot(91, 0, 0)`
    - 크기: `Scale(100.1)`
    - 조작: 'A' 전진, 'Z' 후진 (레버 애니메이션 동기화)

### 수정 금지 원칙
- 상기 **'골든 세팅'**에 명시된 수치와 로직은 사용자의 명시적 요청 없이는 **절대 수정할 수 없습니다.**
- 다른 기능을 추가하거나 리팩토링할 때 해당 로직이 변경될 가능성이 있다면 반드시 사용자에게 경고하고 승인을 얻어야 합니다.

### 자산 배치 및 렌더링 원칙 (LOD 시스템)
- **좌표 기반 배치**: 모든 GLB 모델(지형, 건물, 적 기체 등)은 절대적인 `X, Y, Z` 좌표를 기반으로 배치되어야 합니다.
- **LOD (Level of Detail) 적용**:
    - **원거리 (Distant)**: 실제 GLB 모델을 렌더링하지 않고, 흐릿한 형태(Blurry Shape)나 매우 단순한 플레이스홀더만 표시하여 성능을 최적화합니다.
    - **근거리 (Close-up)**: 플레이어가 접근함에 따라 실제 상세 GLB 모델을 로딩하고 교체하여 보여줍니다.
    - **대상 확장**: 이 원칙은 섬과 같은 환경 자산뿐만 아니라 **적 기체(Enemy Aircraft)의 출현 및 렌더링**에도 동일하게 적용됩니다.

### 기술 스택
- **Engine**: Three.js
- **Bundler**: Vite
- **Language**: JavaScript (ESM)
- **Optimization**: Three.js `LOD` 클래스 또는 거리 기반 동적 로딩 로직 활용
